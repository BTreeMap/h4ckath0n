## 2024-03-03 - Built-in password visibility toggle on primitive Inputs
**Learning:** Component consumers often forget to add proper show/hide visibility toggles onto `<input type="password">` elements during template modification, severely degrading usability and accessibility. Building the visibility toggle directly into the fundamental `<Input>` primitive with correct `aria-label` and `aria-pressed` definitions ensures all downstream implementations inherit these UX best practices out-of-the-box.
**Action:** Always encapsulate basic behavioral form attributes (like password toggles and error bindings) inside design system primitives so that screen readers and casual users benefit globally without relying on the developer remembering to wire up ARIA attributes manually.
