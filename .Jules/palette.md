## 2026-03-10 - [ARIA labels on responsive duplicates] **Learning:** When adding ARIA labels to components that have responsive duplicates (e.g., desktop and mobile navigation toggles), tests relying on `getByRole` will fail because the label makes multiple hidden/visible components match the same accessible name. **Action:** When fixing accessibility on responsive layouts, always check the test suite and convert `getByRole` to `getAllByRole(...)[0]` or use specific test IDs to ensure test resilience.
