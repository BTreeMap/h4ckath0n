## 2024-03-06 - [Universal Form Behaviors in Primitives] **Learning:** [Complex form behaviors like password visibility toggles should be built directly into design system primitive components (e.g., Input.tsx) rather than forcing developers to compose wrapper elements. This ensures automatic, consistent application of accessibility features and interaction patterns across the entire application without relying on developer discipline.] **Action:** [When introducing new interactive form patterns (e.g., clearable inputs, character counters), implement them at the primitive component level instead of creating new wrapper components or expecting developers to compose them manually.]
