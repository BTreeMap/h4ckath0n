## 2026-03-16 - Handling Aria Labels on Responsive Duplicates **Learning:** When adding accessibility features like `aria-label` to components with responsive duplicates (e.g., desktop and mobile toggles in `Layout.tsx`), corresponding unit tests (e.g., `Layout.test.tsx`) must be updated to use `screen.getAllByRole` (and handle arrays) instead of `getByRole` to prevent failures due to multiple element matches. **Action:** Next time I add an accessibility feature to a component that has a responsive duplicate, I will check the tests to make sure that they use `getAllByRole` to prevent multiple match errors.
