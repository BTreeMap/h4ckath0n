## 2024-03-02 - Component-Level Password Visibility Toggle **Learning:** Adding complex interactive behaviors (like password show/hide toggles) at the component primitive level (e.g. `Input`) guarantees automatic accessibility features like `aria-pressed`, focus rings, and input padding are universally applied across all usages without developers needing to remember extra wrapper elements. **Action:** Next time forms have complex universal requirements (like clearable inputs or units), bake the logic into the design system primitive instead of implementing it inline at the consumption level.
