## 2025-06-25 - Safe Pydantic V2 String Normalization
**Learning:** When refactoring duplicated `@field_validator` logic that normalizes strings (like `.strip()`) and raises custom `ValueError` messages, using `BeforeValidator` within `typing.Annotated` is required instead of `AfterValidator`. This ensures normalization happens before Pydantic's built-in constraints (like `max_length`), preventing premature failures. Additionally, because `BeforeValidator` receives unparsed input, the type must be explicitly checked (e.g., `if isinstance(v, str):`) before applying string methods to avoid validation TypeErrors on invalid inputs.
**Action:** When extracting Pydantic validators that modify input prior to constraint checking, always use `BeforeValidator` and include explicit type guards before calling methods on the raw value.
