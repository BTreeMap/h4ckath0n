## 2024-05-24 - Centralize and Extract Functional String Normalization Logic
**Learning:** The h4ckath0n project frequently handles permission strings (scopes) mapped to database text columns. A common anti-pattern was the repetition of ad-hoc filtering, mapping, and deduplication logic for these strings across routers, dependencies, and CLI files. Different files had slightly different implementations (e.g. some using list comprehensions like `[s for s in user.scopes.split(",") if s]`, others using `filter(None, map(str.strip, ...))`). This created correctness risks around whitespace padding and ordering requirements and lacked dedicated unit tests.
**Action:** Created `src/h4ckath0n/auth/scopes.py` with pure helper functions (`parse_scopes`, `format_scopes`, `normalize_scope_list`) to act as the single source of truth for string parsing and formatting logic. This functional decomposition establishes a centralized API interface and facilitates direct unit testing of tricky string parsing semantics, enforcing consistency across application boundaries.
