## 2024-06-13 - Pydantic Field Descriptions and Annotated
**Learning:** When using `typing.Annotated` to centralize Pydantic `Field` validation and constraints (e.g., `DisplayName = Annotated[str, Field(max_length=200, description="..."), AfterValidator(...)]`), the `description` provided inside the alias `Field` will be correctly inherited by the schema, but can be overridden by a new `Field` at the attribute level if needed (e.g., `display_name: DisplayName = Field(..., description="Overrides alias description")`).
**Action:** When extracting shared Pydantic validation into `Annotated` types, include a reasonable default description in the alias `Field` to satisfy common usage, and rely on standard attribute-level `Field` declarations for instances that need context-specific descriptions.
