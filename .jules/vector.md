## 2025-06-21 - Centralizing Pydantic Validator with BeforeValidator
**Learning:** When refactoring duplicated `@field_validator` logic into reusable `typing.Annotated` types in Pydantic V2 to preserve custom `ValueError` messages, `BeforeValidator` is the correct hook if the validation involves normalization (like `.strip()`) that must occur before Pydantic's built-in `Field` constraints (like `max_length`). If we used `AfterValidator`, the input might prematurely fail the `max_length` check because of trailing whitespace that was supposed to be stripped. Furthermore, a `BeforeValidator` receives the raw unparsed input, so we must safely check `isinstance(v, str)` before applying string operations.
**Action:** Always prefer `BeforeValidator` for string normalizations in Pydantic V2 `Annotated` types, and always include type guards in the validator function.
