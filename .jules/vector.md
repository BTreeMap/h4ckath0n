## 2025-02-20 - Centralize validation with Pydantic V2 Annotated and BeforeValidator
**Learning:** When refactoring duplicated string normalization and validation logic (like stripping whitespace and rejecting empty strings) in Pydantic V2, using `Annotated` with a `BeforeValidator` is the idiomatic functional approach. A critical codebase-specific constraint here is that the function passed to `BeforeValidator` receives the raw unparsed input (typed `Any`), so it must explicitly check `isinstance(v, str)` before applying string methods to avoid runtime type errors, allowing non-string inputs to safely fall through to Pydantic's built-in type checkers.
**Action:** Always prefer `Annotated[..., BeforeValidator(...)]` over scattered `@field_validator` classmethods to centralize and share parsing/normalization logic across multiple models, but ensure the validator strictly handles type checking before mutating the value.
