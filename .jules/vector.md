## 2024-05-23 - Centralize List-String Normalization
**Learning:** The codebase heavily uses comma-separated strings for lists (like authentication scopes), which frequently requires parsing, filtering, stripping whitespace, and deduplicating. When this logic is left inline (e.g., using `filter(None, map(str.strip, string.split(",")))`), it is prone to slight inconsistencies (e.g., forgetting to strip whitespace before filtering empty values) and duplication across layers.
**Action:** Always centralize standard list-string normalization into pure helpers (like `parse_scopes` and `format_scopes`) to enforce deterministic parsing, deduplication, and serialization across the codebase. Ensure all new string-list processing uses these centralized helpers rather than ad hoc inline transformations.
