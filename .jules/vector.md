## 2024-07-08 - Scope Normalization and Iterables
**Learning:** `parse_scopes` was designed only for `str`, leading CLI repeated arguments (e.g., `--scope "a,b" --scope "c"`) or variadic API dependencies (`require_scopes("a,b", "c")`) to either break cleanly or require ad-hoc `.extend()` / `,`.join logic scattered at call sites. Python's `argparse.ArgumentParser` stores multiple `--scope` calls as lists of strings, while dependencies expect variadics.
**Action:** When a domain type (like `Scope`) can be constructed from a scalar or a list of scalars (especially common with CLI arguments), push the `str | Iterable[str]` normalization into the lowest level constructor (`parse_scopes`) to ensure a single, pure source of truth rather than duplicating structural normalization pipelines across API endpoints and CLI entrypoints.
