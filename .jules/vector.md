## 2024-03-22 - Centralizing scope string normalization
**Learning:** h4ckath0n manages authorization scopes as a comma-separated string on the user model. Previously, this string was ad-hoc parsed and manipulated using `.split(",")`, `.strip()`, and `filter(None, ...)` across multiple files (`dependencies.py`, `session_router.py`, `cli.py`). This led to repeated code, slight behavioral inconsistencies (e.g. `set()` vs `[]`), and a risk of introducing subtle bugs around whitespaces or duplicates.
**Action:** Created pure functional utilities `parse_scopes` and `format_scopes` in `src/h4ckath0n/auth/scopes.py`. These centralize the normalization logic (handling whitespaces, empty strings, and preserving order while deduplicating). I will prefer using centralized functional parsing helpers for data representation boundaries (like strings to lists) instead of scattering string manipulation.
