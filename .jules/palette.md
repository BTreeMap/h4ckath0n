## 2024-05-14 - Mobile Menu and Layout Accessibility
 **Learning:** When adding responsive icon buttons (like a mobile menu toggle or theme switcher), it's critical to provide explicit `aria-label`s and state management attributes (like `aria-expanded` on menus), otherwise screen readers have zero context. A hidden 'skip to main content' link that becomes visible on focus is essential for keyboard navigation to bypass complex site-wide navigation areas. When modifying UI components that have identical counterparts hidden in responsive modes (e.g. `md:hidden`), automated tests using `getByRole` may begin to fail by finding multiple elements. Tests must be robust enough to handle multiple matched elements (`getAllByRole` vs `getByRole`).
 **Action:** Always audit both desktop and mobile variants of icon-only elements for `aria-label`s, ensure a skip link points to an identifiable `#main-content` target (with `tabIndex={-1}` to receive programmatic focus), and anticipate updating tests to distinguish or select from multiple identical responsive elements.
