## 2025-06-18 - Playwright Testing for Transient React States
**Learning:** When using Playwright to verify transient UI states (like the loading spinner and aria-busy state on a form submission button), typical `getByTestId` approaches may fail if the frontend resets state too quickly or if tests time out waiting for standard UI changes. By intentionally intercepting the network request (e.g., `page.route("**/api/v1/auth/token")`) and injecting an artificial delay with `route.fulfill(...)` or by rendering an isolated version of the HTML component, the Playwright script can reliably pause execution and assert the correct presence of transient ARIA attributes like `aria-busy`.
**Action:** Always intercept API routes to enforce a slow network condition when taking screenshots of or asserting against asynchronous React UI loading states with Playwright.
