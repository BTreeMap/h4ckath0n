## 2026-04-16 - Prevent user enumeration via timing attack on login
**Vulnerability:** The login endpoint returned immediately for non-existent users but performed expensive Argon2id hashing for valid users. This allowed attackers to enumerate registered email addresses by measuring response times.
**Learning:** When mitigating timing attacks using dummy hashes with `argon2-cffi`, the dummy hash must be a structurally valid Argon2id hash string (e.g., starting with `$argon2id$v=19$...`). Passing an invalid string causes `verify_password` to raise an early parsing exception (`argon2.exceptions.InvalidHashError`), which defeats the timing mitigation because the expensive hash computation is skipped. Additionally, the valid hash string must be split across multiple lines using implicit string concatenation to avoid Ruff E501 line length violations.
**Prevention:** Always verify that dummy operations (like password hashing) take roughly the same amount of time as the real operations. Ensure that the dummy hash used for `argon2-cffi` is a fully valid hash string.
